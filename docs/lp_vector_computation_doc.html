計算部（本命）

LP ベクトルの生成

LLM が出力した「商品シーン × 重み」を、スポット空間上の係数ベクトルに変換する。
この係数ベクトルが、ADID ごとの親和性スコア算出のコアになる。

ステップ概要

STEP 1
LLM 出力形式の変換

LLM が返す JSON を (keywords, weights) の平行リストに変換する。
negative シーンは重みを反転（符号をマイナス）することで、ベクトル空間上で「逆方向」を表現する。

INPUT — LLM 出力 JSON
{ "positive": { "コラボカフェの推し色テーブルで 生誕祭の祭壇を即席セット": 0.95, "ライブ会場のグッズ交換エリアで 缶バッジのトレード待機": 0.93, ... }, "negative": { "満員電車の車内で祭壇を広げて 撮影しようとする": 0.98, "ライブ本編中の客席で自立台座 を設置して視界を遮る": 0.97, ... } }
OUTPUT — 平行リスト
lp_keywords = [ "コラボカフェの...", "ライブ会場の...", ..., "満員電車の...", "ライブ本編中の...", ... ] lp_weights = [ +0.95, +0.93, ..., -0.98, -0.97, ... ] # ↑positive はそのまま # ↑negative は符号反転
items_positive = list(analysis_data['positive'].items()) items_negative = list(analysis_data['negative'].items()) lp_keywords = [key for key, _ in items_positive] + [key for key, _ in items_negative] lp_weights = [val for _, val in items_positive] + [-val for _, val in items_negative]
STEP 2
L1 正規化

重みベクトルの絶対値の総和が 1 になるよう正規化する。
positive / negative の比率にかかわらず、重みのスケールを統一するためのステップ。

$$\vec{w}_{L1} = \frac{\vec{w}}{\|\vec{w}\|_1} = \frac{\vec{w}}{\displaystyle\sum_{i=1}^{M} |w_i|}$$
total_weight = np.sum(np.abs(lp_weights)) lp_weights = np.array(lp_weights) / total_weight
なぜ L1 か：L1 ノルムで正規化すると、重みは「割合」として解釈できる。$w_i = 0.3$ なら、キーワード $i$ が LP 全体の意味の 30% を占める。ポジティブ・ネガティブの重みが混在しているため、L2 ではなく L1（絶対値和）で正規化している。
STEP 3
Embedding モデルによる高次元空間への写像

BAAI/bge-m3 モデルで各キーワードを 1024 次元の埋め込みベクトルに変換する。
normalize_embeddings=True により、各行ベクトルは L2 正規化済み（長さ = 1）。

model = SentenceTransformer('BAAI/bge-m3') lp_vector = lp_weights.reshape(1, -1) # 形状: 1 × M lp_matrix = model.encode(lp_keywords, normalize_embeddings=True) # 形状: M × 1024

STEP 4
LP からみたスポットの重要度ベクトル

3 つの行列を連鎖的に掛け合わせ、「この商品がどのスポットと親和性が高いか」を表す 1 × S の係数ベクトルを求める。

$$\underbrace{\vec{lp\_coefficient}}_{1 \times S} \;=\; \underbrace{\vec{lp\_vector}}_{1 \times M} \;\cdot\; \underbrace{lp\_matrix}_{M \times 1024} \;\cdot\; \underbrace{spots\_matrix^\top}_{1024 \times S}$$

この行列積は2 段階に分解して理解するのが本質的である。

前半：重心ベクトルの算出

$$\underbrace{\vec{v}_{\text{centroid}}}_{1 \times 1024} \;=\; \underbrace{\vec{lp\_vector}}_{1 \times M} \;\cdot\; \underbrace{lp\_matrix}_{M \times 1024} \;=\; \sum_{i=1}^{M} w_i \cdot \vec{e}_i$$
なぜ「重心」なのか：$\vec{lp\_vector}$ は L1 正規化済み（$\sum |w_i| = 1$）、$lp\_matrix$ の各行は L2 正規化済み（$\|\vec{e}_i\| = 1$、単位球面上の点）。この組み合わせの行列積は、単位球面上の点群に対する加重平均 = 重心を求める操作に他ならない。

この重心ベクトル $\vec{v}_{\text{centroid}}$ の幾何学的意味は以下の通り：

要素対応役割
$w_i$（L1 正規化済み）LLM が出力した商品シーンの重要度重心計算における重み
$\vec{e}_i$（L2 正規化済み）Embedding モデルが出力した商品シーンの埋め込みベクトル重心計算における位置ベクトル（単位球面上の点）
$\vec{v}_{\text{centroid}}$LP の商品特徴を集約した代表ベクトル重要なシーンの方向に引き寄せられ、negative シーンの方向から遠ざかる
💡 重心ベクトルの大きさが持つ意味：$\vec{v}_{\text{centroid}}$ は L2 正規化されていない。つまり、ベクトルの大きさ（ノルム）が情報を持つ。positive と negative のシーンが似た方向を向いていれば互いに打ち消し合い、ノルムは小さくなる。逆に、positive シーンが一貫した方向を向いている LP は大きなノルムを持ち、後続の計算でスポットとの親和度が強く出る。

COMPUTATION TRACE — リアルタイム計算過程

後半：コホート施設との類似度算出

$$\underbrace{\vec{lp\_coefficient}}_{1 \times S} \;=\; \underbrace{\vec{v}_{\text{centroid}}}_{1 \times 1024} \;\cdot\; \underbrace{spots\_matrix^\top}_{1024 \times S}$$

$spots\_matrix$ の各行ベクトル $\vec{s}_j$ は L2 正規化済み（$\|\vec{s}_j\| = 1$）であるのに対し、重心ベクトル $\vec{v}_{\text{centroid}}$ は正規化されていない。
そのため、この内積は純粋なコサイン類似度ではなく、以下の 2 つの要素を同時に反映する：

$$c_j = \vec{v}_{\text{centroid}} \cdot \vec{s}_j = \|\vec{v}_{\text{centroid}}\| \;\cdot\; \underbrace{\cos\theta_j}_{\text{方向の近さ}}$$
要素数式上の対応意味
重心の大きさ$\|\vec{v}_{\text{centroid}}\|$LP の商品シーンがどれだけ一貫した方向を向いているか。positive/negative が互いに打ち消し合うと小さくなる
方向の近さ$\cos\theta_j$LP の重心方向とコホート施設 $j$ の意味的な類似度
積 = スコア$c_j$LP にとって施設 $j$ がどれだけ重要か。重心に近い施設ほど高スコアになる

行列積の全体像

行列形状正規化意味
lp_vector1 × ML1LLM が出力した商品シーンの重要度（正負付き）
lp_matrixM × 1024L2各商品シーンの埋め込みベクトル（単位球面上の位置）

💡 直感的な理解：この行列積は「LP（ランディングページ）にとって重要なコホート施設はどれか」を、重心からの距離で計算している。LLM が「このシーンに合う」と判断した商品シーンほど重心を強く引き寄せ、その重心に意味的に近いコホート施設が高いスコアを得る。逆に negative シーンは重心を逆方向に押し出すため、negative に近い施設はスコアが低くなる。
STEP 5
L2 正規化

lp_coefficient の方向（各スポットへの相対的な親和度の比率）はそのままに、ベクトルの長さを 1 に揃える。
後続のコサイン類似度計算との整合性を保つため。

$$\vec{lp\_coefficient}_{L2} = \frac{\vec{lp\_coefficient}}{\|\vec{lp\_coefficient}\|_2} = \frac{\vec{lp\_coefficient}}{\sqrt{\displaystyle\sum_{j=1}^{S} c_j^2}}$$
l2norm_vector = np.linalg.norm(lp_coefficient, axis=1, keepdims=True) lp_coefficient = lp_coefficient / np.maximum(l2norm_vector, 1e-10)
np.maximum(..., 1e-10) について：ゼロベクトルでの除算を防ぐガード処理。実運用では全キーワードが同一ベクトルに写像されない限り発生しないが、防御的コーディングとして必須。
CLEANUP
メモリ管理

大規模行列と Embedding モデルを明示的に解放し、GC を起動する。
特に spots_matrix（S × 1024）と model（BGE-M3）は数百 MB〜GB 級のメモリを占有するため、不要になった時点で即座に開放する。

del spots_matrix del model, lp_matrix del l2norm_vector gc.collect()

統合数式

全ステップを 1 行にまとめると：

$$\vec{lp\_coefficient} = \text{normalize}_{L_2}\!\left(\;\frac{\vec{w}_{\text{raw}}}{\|\vec{w}_{\text{raw}}\|_1} \;\cdot\; \text{BGE-M3}(\text{keywords}) \;\cdot\; S^\top\;\right)$$

Shape の変化を追うと：

$$\underbrace{\vec{w}}_{1 \times M} \;\cdot\; \underbrace{E}_{M \times 1024} \;\cdot\; \underbrace{S^\top}_{1024 \times S} \;=\; \underbrace{\vec{c}_{\text{raw}}}_{1 \times S} \;\xrightarrow{\;L_2\;}\; \underbrace{\vec{c}}_{1 \times S}$$
